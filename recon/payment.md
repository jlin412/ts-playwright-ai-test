## Key behavioral facts

1. **PAY button starts disabled** and enables only when Stripe considers the form valid (all required card fields filled + email provided).
2. **Stripe iframes are accessible** via Playwright `frameLocator`. The first `__privateStripeFrame` contains all 4 card fields rendered by Stripe's Card Element.
3. **hCaptcha fires** (`api.hcaptcha.com`) when a valid email is present and PAY is clicked. This blocks actual charges — safety net.
4. **Email validation**: Filling "notanemail" + attempting submit → "Not a valid email address" visible in DOM. HTML5 validity `valid: false`.
5. **Production Stripe keys** — confirmed by live `r.stripe.com/b` metrics. Test card numbers with actual submission would create real charges if captcha is bypassed.

## How to fill Stripe fields safely

- **Fill partial data only** (expiry, CVC, ZIP without a valid card number) — PAY stays disabled, no charge attempt
- **Never fill a complete valid card number** in the `__privateStripeFrame` against production keys
- Use Stripe's publicly-documented test PAN format `4242 4242 4242 4242` ONLY in test-mode environments
