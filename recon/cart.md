## TL;DR

The Cart tab is **view-only** for guests. There are NO inline quantity controls, NO per-line remove button. Reload **clears the cart** and redirects to `/`. Workflow M scope has to shrink to match.

## What's missing on the Cart tab

- **No +/- quantity adjusters** on the line item
- **No "Remove" button** on the line item (only "EMPTY CART" exists, and that's on the **Tickets** tab вЂ” not Cart)
- **No edit mode**

To change quantity, the user has to navigate **back to the Tickets tab** (or click another `ADD` on the same ticket type before advancing to Cart).

## Persistence behavior (important contract)

- Reload of `/checkout/cart`:
  - URL **redirects to `/`** (the home/landing page renders)
  - Cart shows `Your Cart is Empty`, `Total $0.00`
  - `NEXT: PAYMENT` button becomes disabled
- Fresh context visiting `/checkout/cart` directly:
  - Same result: redirect to `/`, cart empty
- **localStorage** only stores `acceptedCookieNotification`; no cart state
- **Cookies** include `ahoy_visit`, `ahoy_visitor`, `circuit_id`, `site_id` вЂ” but these don't reseed the cart on reload

The cart is held **purely in Vue state in memory**. Lose the page в†’ lose the cart. This is the contract; if it ever changes, our test should flag it.
