# Reconnaissance findings — yosemitecinema.com

Generated by [run-recon.mjs](run-recon.mjs), [movie-detail-recon.mjs](movie-detail-recon.mjs), and [showtime-click-recon.mjs](showtime-click-recon.mjs). Re-run any script to refresh. Screenshots in `recon/screenshots/`.

## Headline finding

**Everything is same-origin.** Ticketing, seat selection, food & drink, cart, and payment are all served by yosemitecinema.com — no Veezi, no Fandango, no third-party iframes. The site is a Vue.js SPA using the Quasar UI framework (button class `q-btn`).

All extended workflows (K–O from the test plan) are fully automatable against the live origin. **Do not need a staging environment** for negative-path testing; use `Continue As Guest` and **never submit a real card**.

## Ticketing & checkout flow (verified)

```
/now-playing
  ↓ click <a href="/movie/{id}">
/movie/{id-or-slug}
  ↓ click <button>{HH:MM AM|PM}</button>      ← same-tab navigation
/checkout/showing/{slug}/{showtime-id}
  ↓ tab pills: Tickets | Food & Drink | Cart | Payment
  ↓ auth prompt: Log In / Sign Up / Continue As Guest
```

Verified concretely: clicking "12:45 PM" on `/movie/7579` (Experience Yosemite VR) → `/checkout/showing/experience-yosemite/1021546`. Same tab, no popup, no cross-origin host.

### Tickets tab
- Movie metadata strip at top (title, audience rating, genre, runtime, date, time range, theater "Yosemite Cinema", screen "Yosemite VR")
- Auth panel: `Email` + `Password` + `Log In` button, `Sign Up` button (Hero Points reward system), `Continue As Guest` button
- Cart panel: `Your Cart is Empty` + Promo code input (`aria-label="Add Gift Card, Voucher, Promo Code"`) + `APPLY` button
- Ticket type selectors (Adult / Child / Senior etc.) — not yet inspected; likely revealed after `Continue As Guest`

### Food & Drink tab
- Confirmed via `/checkout/items` direct visit
- Categories: **Popcorn, Beverages, Beer and Wine, Snacks, Treats, Food**
- Each item: name, optional description, `$price`, `Add` button
- `AGE RESTRICTED` warning on Beer/Wine section
- Item modifiers TBD — every item just shows an "Add" button at top level; tapping likely opens a modifier modal

### Cart tab
- Reachable via `NEXT: CART` button
- Starts as `Your Cart is Empty`
- Promo code field persists across tabs

### Payment tab
- Not yet inspected
- **Do not submit real payments during testing**

## What's volatile vs. stable

**Stable (assertable):**
- Route patterns (`/now-playing`, `/movie/*`, `/checkout/showing/*/*`, `/checkout/items`)
- Tab labels (`Tickets`, `Food & Drink`, `Cart`, `Payment`)
- Auth panel labels (`Log In`, `Sign Up`, `Continue As Guest`)
- Concession category names (Popcorn, Beverages, Beer and Wine, Snacks, Treats, Food)
- Promo code aria-label (`Add Gift Card, Voucher, Promo Code`)
- Membership tier names + prices on `/membership--guest-pricing` (Diamond Elite, Platinum, Unlimited Movie, Day Pass $18)
- Header nav (Showtimes → /calendar, Coming Soon → /coming-soon, Virtual Reality, Members)

**Volatile (use structural checks):**
- Movie titles, posters, showtimes — rotate weekly
- Per-item concession prices ($6.00 popcorn, $10.00 caramel corn, etc.) — soft-check
- Whether a given film has any scheduled showtimes (e.g., Toy Story 5 currently has none)
- Specific showtime IDs (e.g., `1021546`)

## Implications for Workflows K–O

Original plan assumed third-party hosting. **All five can be implemented same-origin against production**, with these scope adjustments:

### Workflow K — Movie ticket purchase ✅
- Implementable end-to-end. Pick first movie *with showtimes* (skip "Nothing Scheduled"). Click first time button → land on `/checkout/showing/*/*`. Use `Continue As Guest`. Validate ticket-type selectors render. **Stop before Payment submission** (route-mock the POST to verify the UX without charging).

### Workflow L — Food & Drink ✅
- Navigate to `/checkout/items` directly OR reach via Tickets→Food tab. Add an item, verify modifier modal (if present), verify quantity controls, verify cart line.

### Workflow M — Cart ✅
- Reach via `NEXT: CART`. Validate totals, line removal, quantity update. Promo-code field persists — verify it carries across tab switches.

### Workflow N — Checkout validation ✅
- Reach Payment tab. Run empty-submit, invalid-email, invalid-card, declined-card tests. **Use Playwright `page.route()` to intercept the payment POST** and fail it — confirms the UX without ever submitting to the real processor.

### Workflow O — Membership / discounts ⚠️ scope reduced
- `/membership--guest-pricing` is informational only — **no online subscribe CTA exists**. Drop O.2 (membership purchase entry).
- Promo code testing (O.5–O.8) IS fully implementable on the cart's promo-code field.
- Member discount on ticket pricing (O.3) requires a real member login — defer until we have a test account.
- Hero Points sign-up flow exists (`Sign Up` button on Tickets tab) — could test signup form validation against `route()`-mocked submit, but not a real registration.
